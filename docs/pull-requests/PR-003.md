## 🔍 Cambios Técnicos Destacados

### Validación en tiempo real
```javascript
entrada.addEventListener('blur', () => validarCampo(entrada));
entrada.addEventListener('input', () => {
  if (entrada.classList.contains('invalido')) validarCampo(entrada);
});
```

La validación se activa:
1. Al salir de un campo (`blur`): siempre valida.
2. Al escribir en un campo (`input`): solo si el campo ya fue marcado como inválido.

### Patrón IIFE
Todo el código JavaScript del cliente está envuelto en un IIFE para evitar variables globales:

```javascript
(function () {
  'use strict';
  // Código del cliente...
})();
```

### Contador de caracteres
El contador cambia de color a rojo cuando el usuario supera el 90% del límite de caracteres, proporcionando una advertencia visual anticipada.

**@andres-villa:** Consideré esa opción, pero decidí no validar en cada tecla para los campos aún no visitados, ya que generaría errores prematuros (por ejemplo, mostrar "correo inválido" mientras el usuario está a mitad de escribirlo). El evento `blur` + `input` condicional es un buen equilibrio entre reactividad y experiencia de usuario.

**@sofia-ramirez:** Tienes razón, es un buen balance. Aprobado. El patrón IIFE es una buena decisión para el código del cliente.
